Use low stringency during SNP analysis?
By default, Easymap only considers SNPs that pass high quality thresholds. However, in sub-optimal read datasets, more lenient filtering can help to identify a candidate interval and the causal mutation. If you analyzed your reads in the default mode and obtained very few SNPs or believe that the causal mutation could have been discarded, turn on this option and run the program again.
Perform automated pre-processing of read files?
Checking this option will add an optional step to process the input reads to eliminate low-quality data, trim low-quality nucleotides and auto-remove adapters. This step is performed with fastp and will require additional storage space.

Use multiple CPU cores during analysis?
Using multiple cores will notably accelerate certain parts of the analysis. If you dont know how many cores your computer can spare, leave this option unchecked, otherwise please indicate how many cores Easymap can use.
